ML Algorithms in ANN flow

Simple Machine Learning algorithms based on the flow of Artificial Neural Network. The layer(s) used in the algortihms are dense type.

Overview

The project is made in Scala purely with Nd4J(equivalent to Numpy in Python), a scientific computing library for the JVM with routines designed to run fast with minimum RAM requirements. It is a typical sbt project.

Please follow the below steps to setup and use the same:

  1. Install and setup sbt.
  2. Download or clone the repository.
  3. Open terminal at the downloaded(or cloned) directory.
  4. Command: sbt run and specify the class you want to run.

Note

  • In order to tune an algorithm for a dataset, play around with hyperparameters specified for each algorithm.
  • It may be required to implement initialization and data transformation techniques for better results, but the sole purpose of the project is to depict the way algorithms work in an ANN.

Each algorithm designed follows the phases defined below using Vector Algebra, Calculaus and Matrix Manipulations. The core equations remains same but there are derived equations as the algortihm changes.

Algorithm Flow

For number of samples $m$, features $n$ in an input matrix $X_{m\times n}$, $f_{act}$ as Activation function, $W_{1\times n}$ as Weight vector, $B$ as Bias, assuming $i$ ranging from $1$ to $m$ and $j$ ranging from $1$ to $n$, an ANN model is governed by:

  • Output: $\hat{y_i}=\sum_{j=1}^n z_i$
  • $z_i = (W_{1\times n}). (x_{1\times n}) + B$
  • Cost: $J(\hat{Y}, Y)$

Each algorithm follows the below stages.

Forward Propagation

The process by which an ANN produces output is governed by each component neuron. Activation function is one of the factors that changes as an Algorithm changes.

Cost - An equation to penalize the model and using the penalty to train the model. The cost function is always made such that it should be convex in nature, hence while performing Gradient Descent, the Cost should converge to the global minima. Typically for :

  • Classification:
    • SVM: $J(\hat{Y},Y)=\frac{\lambda}{2}\parallel W\parallel^2 + \sum_{i=1}^m max(0,1-y_i\times\hat{y_i})$
    • Others: $J(\hat{Y},Y)=\sum_{i=1}^m y_i\times log_e(\hat{y_i})$
  • Regression: $J(\hat{Y},Y)=\frac{1}{2\times m}\sum_{i=1}^m (\hat{y_i}-y_i)^2$

Backward Propagation

The process of learning and updating the learnable parameters in order to achieve maximum possible accuracy and minimum possible cost(or loss). The whole process is governed by Gradient Descent that minimizes the Cost function by taking specified number of steps to reach a global minima. Gradient Descent is evaluated by chain rule as :

  • Weights Gradient:

    $\frac{\partial J(\hat{y_i},y)}{\partial w_j}=\frac{\partial J(\hat{y_i},y)}{\partial \hat{y_i}}\times\frac{\partial \hat{y_i}}{\partial z_i}\times\frac{\partial z_i}{\partial w_j}$

  • Bias Gradient:

    $\frac{\partial J(\hat{y_i},y)}{\partial B}=\frac{\partial J(\hat{y_i},y)}{\partial \hat{y_i}}\times\frac{\partial \hat{y_i}}{\partial z_i}\times\frac{\partial z_i}{\partial B}$

The dependency comes due to:

  • $J=f(\hat{y_i})$
  • $\hat{y_i}=f_{act}(z_i)$
  • $z_i=g(w_j,B)$

Parameters Update

As learnable parameters keep updating to achieve a desired accuracy, the model keeps learning. This is gauged by another hyperparameter - learningRate, that acts as a step(large or small) scaled of the descent to reach the global minima.

  • When kept too large, it bounces back and forth off the global minima, hence always passing forth the global minima.
  • When kept too small, it enables many descents to reach the global minima thereby slowing down the learning. Hence the parameter should be adjusted accordingly.

The parameters are updated as :

  • $W_{new}=W_{old}-\alpha\times \frac{dJ}{dW}$

  • $B_{new}=B_{old}-\alpha\times \frac{dJ}{dB}$

    where:

    • $\frac{dJ}{dW}=\sum_{j=1}^n \frac{\partial J(\hat{y_i},y)}{\partial w_j}$
    • $\alpha$ is Learning Rate
