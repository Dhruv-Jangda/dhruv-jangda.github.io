ML Algorthms in ANN Model

Simple Machine Learning algorithms based on the flow of Artificial Neural Network. The layer(s) used in the algortihms are dense type.


Algorithm Flow

For number of samples m, features n in an input matrix Xmxn, fact as Activation function, W1xn as Weight vector, B as Bias, assuming i ranging from 1 to m and j ranging from 1 to n, an ANN model is governed by:

\[output:\hat{y_i}=\sum_{j=1}^n z_i\] \[z_i = (W_{1\times n}). (x_{1\times n}) + B\] \[cost:J(\hat{Y}, Y)\]
  1. Forward Propagation - The process by which an ANN produces output is governed by each component neuron. Activation function is one of the factors that changes as an Algorithm changes.

  2. Cost - An equation to penalize the model and using the penalty to train the model. The cost function is always made such that it should be convex in nature, hence while performing Gradient Descent, the Cost should converge to the global minima. Typically for :
    • Classification:
      \[SVM:J(\hat{Y},Y)=\frac{\lambda}{2}\parallel W\parallel^2 + \sum_{i=1}^m max(0,1-y_i\times\hat{y_i})\] \[Others:J(\hat{Y},Y)=\sum_{i=1}^m y_i\times log_e(\hat{y_i})\]
    • Regression:
      \[J(\hat{Y},Y)=\frac{1}{2.m}\sum_{i=1}^m (\hat{y_i}-y_i)^2\]

  3. Backward Progagation - The process of learning and updating the learnable parameters in order to achieve maximum possible accuracy and minimum possible cost(or loss). The whole process is governed by Gradient Descent that minimizes the Cost function by taking specified number of steps to reach a global minima. Gradient Descent is evaluated by chain rule as :
    • Weights Gradient :
      \[\frac{\partial J(\hat{y_i},y)}{\partial w_j}=\frac{\partial J(\hat{y_i},y)}{\partial \hat{y_i}}\times\frac{\partial \hat{y_i}}{\partial z_i}\times\frac{\partial z_i}{\partial w_j}\]
    • Bias Gradient :
      \[\frac{\partial J(\hat{y_i},y)}{\partial B}=\frac{\partial J(\hat{y_i},y)}{\partial \hat{y_i}}\times\frac{\partial \hat{y_i}}{\partial z_i}\times\frac{\partial z_i}{\partial B}\]
    The dependency comes due to:
    \[J=f(\hat{y_i})\] \[\hat{y_i}=f_{activation}(z_i)\] \[z_i=f(w_j,B)\]
  4. Parameters Update - As learnable parameters keep updating to achieve a desired accuracy, the model keeps learning. This is gauged by another hyperparameter - learningRate, that acts as a step(large or small) scaled of the descent to reach the global minima. When kept too large, it bounces back and forth off the global minima, hence always passing forth the global minima. However, when kept too small, it enables many descents to reach the global minima thereby slowing down the learning. Hence the parameter should be adjusted accordingly. The parameters are updated as :
    \[W_{new}=W_{old}-\frac{dJ}{dW}\] \[B_{new}=B_{old}-\frac{dJ}{dB}\] where: \[\frac{dJ}{dW}=\sum_{j=1}^n \frac{\partial J(\hat{y_i},y)}{\partial w_j}\]
