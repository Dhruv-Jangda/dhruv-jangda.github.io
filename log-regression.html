Logistic Regression

Generalized Linear Models

  • Dhruv Jangda
  • Logistic Regression

GLMs are extension of the linear model framework with DV both continuous(Normal distribution) and discrete(Non-Normal distribution).

  1. Comprise a linear combination of input features.
  2. DV should have PDF(Probability Distribution Function) belonging to the family of exponential distribution:
    • Binomial - when DV is binary
    • Poisson - when DV is multi-class
    • Gaussian - when DV is continuous

Logistic Regression Types

  1. Multinomial Logistic Regression
    • For DV with $K$ classes, the technique fits $K-1$ independent binary logistic classifier models.
    • It randomly chooses a class as the reference for each $K-1$ regression models and fits them.
    • Isn't used widely as it does not scale very well when $K$ is large enough.
    • Since it builds $K-1$ models, a larger dataset is required to achieve a reasonable Accuracy.
  2. Ordinal Logistic Regression
    • Used when DV is ordinal in nature i.e. order matters or sequential i.e. 5>4>3>2>1 as an example.
    • For DV with K classes, the technique builds a single model with K-1 threshold values.
    • It makes an imperative assumption of proportional odds i.e. on a logit (S shape) scale, all thresholds lie on a straight line.

Model

Though most of the underlying assumptions are same as of Linear Regression, but DV being discrete, below can be taken to account:

  1. Each trial(or row) can have only two outcomes - success or failure.
  2. Probability of success p and failure q must be same for each trial.

Since, logistic regression model is made on basis on linear regression model, hence:

$y_{predicted}: \hat{y} = \beta_0 + \vec{\beta}.\vec{X} + \epsilon$

But due to DV being discrete, $y_{predicted}$ or $\hat{y}$ needs to be converted to probability and a threshold has to be selected to conclude with outcomes of success or failure.

Hence, probability of success is defined as:

$P(y=1|X): p = sigmoid(y_{predicted}) = \frac{e^\hat{y}}{1 + e^\hat{y}}$

Rearranging the equation:

$\log(\frac{p}{1-p}) = \hat{y} = \beta_0 + \vec{\beta}.\vec{X} + \epsilon$

The term $\log(\frac{p}{1-p})$ is called logit or $\log(odds)$ where $odds$ in probability is $\frac{P(success)}{P(failure)}$. Since, each trial can be either success or failure, so $P_{failure} + P_{success} = 1$

Solution to Model

Likelyhood

As observed from above equation that $\log(odds)$ simply gives a line.

Now the $\log(odds)$ is converted to probability by passing it through the sigmoid function $\frac{e^\hat{y}}{1 + e^\hat{y}}$. Then converting the probabilities to that of their respective classes i.e. for success $p$ when $y=1$ for failure $1-p$ when $y=0$

The likelyhood function i.e. the product of all probabilities of the corresponding class is given by: $l(\beta_i) = \prod_{i,y_i=1} p(x_i) \times \prod_{i,y_i=0} (1-p(x_i))$

And the value of $\beta_i$ for which the log likelyhood maximises is called maximum likelyhood estimator and is the solution to the likehood equation:

$\frac{\partial(log(P(X|\beta_i)))}{\partial(\beta_i)} = 0$

Hence the maximum likelyhood tries to find the value of coefficients $\beta_i$ such that the predicted probabilities are as close to the observed probabilities as possible.

Observe the tranformation from the linear equation i.e. the $\log(odds)$ to the likelyhood estimation, whose log when maximized gives the optimim value $\beta_i$ for a feature $i$

Objective Function

For linear regression, the objective function with cost as MSE simply is:

$J(\beta_i) = \frac{1}{m}\times[\sum_{i=1}^{m} (y_i - \hat{y_i})^2] + \lambda\times L_i$

where:

  • $m$ is the number of samples and $n$ is the number of IDVs or features.
  • $\lambda$ is the regularization parameter and helps generalizing the model i.e. preventing overfitting.
  • $i$ is for 1 or 2 representing L1(Lasso Regression) or L2(Ridge Regression) regularization:
    • $L2 = \sum_{j=1}^{n} \beta_j^2$
    • $L1 = \sum_{j=1}^{n} |\beta_j|$

To just keep, things simple considering just MSE for logistic regression, gives a function with multiple minimas, hence not qualifiying for the definition of a Convex function.

Hence, for logistic regression the cost function is:

$ cost(y_i, \hat{y_i})= \left\{ \begin{array}{ll} -\log(\hat{y_i}) & y_i=1\\ -\log(1-\hat{y_i}) & y_i=0\\ \end{array} \right. $

And the overall objective function $J(\beta_j)$ includes regularization as well, where $j \in [1,IDV]$ and $i \in [1,samples]$.

As per above cost function:

  • For $y_i=1$

  • For $y_i=0$

The combined cost function is:

$cost(y_i, \hat{y_i}) = -\frac{1}{m} \times \sum_{i=1}^{m}[y_i\times\log(\hat{y_i}) + \bar{y_i}\times\log(1-\hat{y_i})]$

where

  • $\hat{y_i}$ represents the probability of success i.e. $sigmoid(y_{predicted}) = \frac{e^\hat{y}}{1 + e^\hat{y}}$
  • Negative sign came due to the fact that $\hat{y_i} \in [0,1]$ and so $\log(\hat{y_i}) \in (-\infty,0]$
  • Also observe as per above graphs when $y=1$:
    • $\hat{y_i}=0$, the cost function gives $-(-\infty)=+\infty$
    • $\hat{y_i}=1$, the cost function gives $-(0)=0$

Accessing the fit of Model

  1. Akaike Information Criteria (AIC)
    • Counterpart of R-squared in linear regression.
    • Hence, smaller is better.

  2. Confusion Matrix

Confusion matix is the most important metric of any classification problem. It gives following metics:

  1. Accuracy = $\frac{TP+TN}{n}$

    • General metric
    • Given preference when the data is balanced.

  2. Precision = $\frac{TP}{TP + FP}$

    • The metric is based on Predicted values by the model(observe P common Precision/Predicted to remember).
    • Given preference when FP in a model are harmful. Eg: A model predicts that the transaction done is normal although it was fradulent.

  3. Recall = $\frac{TP}{TP + FN}$

    • The metric is based on Actual values observed by the model.
    • Also called Senitivity or True Positive Rate (TPR) i.e. the proportion of actual positives predicted correctly by the model and is $1-FNR$.
    • Given preference when FN in a model are harmful. Eg: A model predicts that the patient has no cancer although in reality he has.

  4. F score: $F_\beta = \frac{(1+\beta^2)\times(Precision \times Recall)}{\beta^2 \times Precision + Recall}$

    • $\beta$ represents the balance between the Precision and Recall.
    • Given preference based on which of Precision or Recall holds more importance
      • $\beta = 1$ implies both Precision & Recall holds equal importance.
      • $\beta < 1$ implies Precision holds more importance than Recall.
      • $\beta > 1$ implies Recall holds more importance than Precision.

  5. ROC Curve

    • The metric is based on TPR(True Positive Rate) and FPR(False Positive Rate)
    • The Area under this Curve or AUC gives the idea of the best fit model and should be close to 1.
    • FPR is the proportion of actual negatives incorrectly predicted. $FPR = \frac{FP}{FP + TN} = 1 - TNR$
    • The balance between TPR and FPR changes by change in the threshold value $\lambda$. The value of $\lambda$ is set based on this curve and as per the domain.
    • Hence for different values of $\lambda$ different values of TPR and FPR are plotted.
